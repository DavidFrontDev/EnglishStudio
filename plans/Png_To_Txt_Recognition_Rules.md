# Правила конвертации PNG в TXT для IELTS Reading

## Общий принцип

1. Распознавать текст с изображения вручную по содержимому PNG.
2. Не использовать Windows OCR.
3. Сохранять результат в `.txt` рядом с исходным `.png`.
4. Кодировка файла: UTF-8.
5. Сохранять исходный текст задания максимально близко к изображению: заголовки, номера вопросов, инструкции, варианты ответов, названия таблиц и списков.

## Именование файлов

1. Для одиночного изображения создавать `.txt` с тем же базовым именем:
   - `Anketa 2.png` -> `Anketa 2.txt`
   - `Radio 6.png` -> `Radio 6.txt`
   - `Comparison 9 картинка.png` -> `Comparison 9 картинка.txt`
2. Пробелы, кириллицу и дополнительные слова в имени сохранять без изменений.
3. Если `.txt` уже существует, его можно перезаписать исправленной версией.

## Многочастные изображения

1. Если несколько PNG являются частями одного текста и имеют суффиксы вида `_1`, `_2`, `_3`, их нужно объединять в один `.txt`.
2. Имя итогового файла должно быть без суффикса части:
   - `text 1_1.png` + `text 1_2.png` -> `text 1.txt`
   - `part 2_1.png` + `part 2_2.png` -> `part 2.txt`
3. Части объединять по возрастанию номера суффикса: `_1`, затем `_2`, затем `_3`.
4. Не создавать отдельные `.txt` для файлов-частей вроде `part 1_1.txt` или `text 2_2.txt`.
5. При объединении сохранять естественный порядок чтения. Если первая часть обрывается на середине предложения, продолжать текст со второй части без искусственного заголовка между частями.

## Буквенные параграфы и секции

1. Если в тексте passage есть буквенные обозначения параграфов или секций (`A`, `B`, `C` и т.д.), ставить букву перед первым предложением этого параграфа в формате:
   - `A. The automotive sector...`
   - `E. Automation may...`
2. Не оставлять букву отдельной строкой перед параграфом.
3. Это правило применяется только к параграфам/секциям passage.
4. В списках вариантов ответов буквы, римские цифры и другие обозначения вариантов сохранять с точкой после обозначения, даже если на изображении точки нет:
   - `A. Peter Fleming`
   - `B. Ran Fiennes`
   - `i. The areas and artefacts within the pyramid itself`
   - `A. may be provided through methods used in biological research.`

## Форматирование текста

1. Заголовки и инструкции оставлять отдельными строками.
2. Между крупными блоками текста оставлять пустую строку.
3. Номера вопросов сохранять перед текстом вопроса с точкой после номера:
   - `14. reference to ...`
   - `37. What method did ...?`
4. Пропуски для ответов передавать как `................`.
5. Маркеры списков можно передавать дефисами:
   - `- Moore's exhibition ...`
6. Таблицы распознавать построчно, разделяя столбцы через ` | `.
   - Заголовки столбцов писать в одной строке: `Part of tree | Traditional use`.
   - Каждую строку таблицы писать в одной строке: `6 ................ | fuel`.
   - Если в одной ячейке несколько пропусков, сохранять их внутри этой ячейки: `7 ................ and ................ | medicine`.
   - Не выравнивать столбцы пробелами вместо `|`.
7. Сноски сохранять рядом с passage, обычно после абзаца или блока, к которому они относятся.
8. Кавычки, апострофы, проценты, даты, имена и специальные термины сохранять точно по смыслу изображения.

## Файлы с картинками-списками

1. Если PNG содержит только список вариантов, список людей, список headings или другой вспомогательный блок, создавать отдельный `.txt` с тем же именем.
2. Примеры:
   - `Comparison 5 картинка.png` -> `Comparison 5 картинка.txt`
   - `Selector_Image 5 картинка.png` -> `Selector_Image 5 картинка.txt`
3. Такой файл не объединять с основным заданием, если в имени нет суффикса части `_1`, `_2`.

## Контроль качества

1. Перед сохранением просматривать изображение в оригинальном качестве.
2. После записи проверять, что `.txt` создан рядом с `.png`.
3. Для папки сверять итоговый список: у одиночных PNG должен быть TXT с тем же именем, у многочастных PNG должен быть один объединенный TXT.
4. Если обнаружена ошибка форматирования, исправлять только соответствующий `.txt`, не меняя остальные файлы.
